===============================================
KIANGOMBE HEALTH CENTER - DATABASE SUMMARY
===============================================

OVERVIEW:
---------
Comprehensive health center management system with 18 main tables handling:
- User management (patients, doctors, staff)
- Appointment scheduling and management
- Medical records and history tracking
- Prescription management
- Pharmacy inventory
- Payment processing and billing
- Activity logging and security

TABLE COUNT: 18 tables
RELATIONSHIPS: 25+ foreign key relationships
INDEXES: 15+ performance indexes
CONSTRAINTS: 10+ validation rules

===============================================
TABLE STRUCTURE OVERVIEW
===============================================

CORE ENTITIES (6 tables):
--------------------------

1. USERS (Central table)
   - Handles all user types: patients, doctors, nurses, etc.
   - Role-based access control
   - Authentication and profile management

2. DOCTORS, NURSES, RECEPTIONISTS, LAB_TECHNICIANS, PHARMACISTS
   - Staff profile tables (one-to-one with users)
   - Professional details and specializations
   - Availability and licensing information

3. APPOINTMENTS
   - Central scheduling system
   - Links patients to clinicians
   - Payment status integration
   - Visit type tracking (in-person, video, medication_purchase)

BUSINESS LOGIC (4 tables):
--------------------------

4. PRESCRIPTIONS
   - Medical prescriptions with JSON medication details
   - Status tracking (pending, approved, fulfilled)
   - QR code support
   - Pharmacy integration

5. MEDICATIONS
   - Pharmacy inventory management
   - Stock tracking and expiry dates
   - Pricing and supplier information
   - Batch number tracking

6. PAYMENTS
   - Financial transaction processing
   - M-Pesa integration support
   - Multiple payment methods
   - Transaction ID tracking

7. MEDICAL_HISTORY
   - Comprehensive patient health records
   - Diagnosis, symptoms, treatment tracking
   - Attachment support via JSON
   - Doctor-patient relationship tracking

PATIENT PROFILES (4 tables):
--------------------------

8. MEDICAL_INFO
   - Patient health details (blood type, height, weight)
   - Allergies and conditions (JSON format)
   - Current medications tracking

9. EMERGENCY_CONTACTS
   - Emergency contact information
   - Relationship tracking

10. INSURANCE
    - Insurance provider details
    - Policy numbers and coverage limits
    - Quarterly usage tracking

11. NOTIFICATION_SETTINGS, SECURITY_SETTINGS
    - Patient preference management
    - Two-factor authentication support
    - Communication preferences

STAFF MANAGEMENT (2 tables):
---------------------------

12. DOCTOR_AVAILABILITY
    - Weekly scheduling patterns
    - Time slots and break periods
    - Appointment duration settings

13. DOCTOR_SETTINGS
    - Profile visibility preferences
    - Notification preferences
    - Booking availability settings

SUPPORT SYSTEMS (2 tables):
--------------------------

14. ACTIVITY_LOGS
    - User activity tracking
    - Security audit trail
    - Device and location logging

15. WISHLISTS
    - Patient medication favorites
    - Many-to-many relationship support

===============================================
KEY RELATIONSHIPS
===============================================

CENTRAL HUB: Users table
├── One-to-One: Profile tables (medical_info, emergency_contacts, etc.)
├── One-to-Many: Appointments (as patient)
├── One-to-Many: Appointments (as clinician)
├── One-to-Many: Medical History (as patient)
├── One-to-Many: Medical History (as doctor)
├── One-to-Many: Payments (as patient)
├── One-to-Many: Payments (as doctor)
└── One-to-Many: Activity Logs

APPOINTMENT SYSTEM:
Appointments → Prescriptions (one-to-one optional)
Appointments → Payments (one-to-many)
Appointments → Medical History (one-to-many)

PHARMACY SYSTEM:
Medications → Prescriptions (one-to-many)
Medications → Wishlists (one-to-many)

===============================================
PERFORMANCE CONSIDERATIONS
===============================================

INDEXES IMPLEMENTED:
- Primary keys on all tables
- Foreign key indexes for joins
- Search indexes: email, phone, names
- Date indexes: appointments, medical_history
- Status indexes: appointments, payments
- Business indexes: medications.category, doctors.specialization

QUERY OPTIMIZATION:
- Patient appointment lookup: indexed on patient_id + scheduled_at
- Doctor schedule: indexed on clinician_id + scheduled_at
- Payment tracking: indexed on transaction_id
- Medical history search: indexed on patient_id + date

SCALABILITY FEATURES:
- JSON fields for flexible data storage
- Enum fields for data integrity
- Timestamp fields for auditing
- Status fields for workflow management
